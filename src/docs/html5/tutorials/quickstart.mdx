## Step 3: Modules

The example project comes with preconfigured config files `rivet.json` and `rivet.dev.json`.

- **`rivet.json`**

    All configuration regarding [modules](/docs/html5/general/modules/quickstart) goes in this file.
- `rivet.dev.json`

    This file is used to override properties from `rivet.json` for local development.

## Step 4: Server

The example project's backend server exists in the `game_server` folder.

- The `game_server.Dockerfile` dockerfile copies this folder and builds a Docker image from it.
- This image is uploaded to Rivet allow running game servers remotely anywhere in the world.

## Step 5: Run locally

This will build & run your backend locally & hot reload when you make changes to the configuration.

```sh
./scripts/run_local.sh
```

1. Navigate to http://localhost:8080/ to view the running example.
2. Choose `Find or Create Lobby` to connect to the backend.

## Step 6: Deploy

Deploy the example to Rivet:

```sh
./scripts/deploy.sh
```

Then, start the client locally via:

```sh
./scripts/run_remote.sh
```

1. Navigate to http://localhost:8080/ to view the running example.
2. Choose `Find or Create Lobby` to connect to the backend through Rivet.
